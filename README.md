# AI Capital Cycle — Multi-Method Bubble Diagnostics

**Erika Francesca Caragnano - ESCP Business School / Politecnico di Torino**

This repo contains a pipeline for detecting and date-stamping speculative episodes in
AI-exposed equities using ADF / PP / KPSS / SADF / GSADF (PSY 2015) /
SV-ADF (Sarkar & Wells 2026). Universe: ~80 tickers organized by position in
the AI value chain taxonomy developed by the author. 

---

## Highlights

- **Incremental Monte Carlo cache** — add or remove a ticker and only new
  simulations run. An interrupted run resumes from where it left off.
- **Selective method runs** — `--methods svadf` to re-run only SV-ADF, or any
  combination, without touching the cached MC.
- **Window-flexible SV-ADF** — runs at a user-chosen window per call; multiple
  windows per ticker coexist on disk side-by-side.
- **Spurious-bubble filters** — `EPISODE_MIN_DURATION` and `EPISODE_MERGE_GAP`
  defaults prevent micro-episodes from Monte Carlo noise.
- **Data ≠ plots** — results are stored as JSON + Parquet under `data/results/`.
  The plotting layer reads from disk; rewriting figures never re-runs analysis.
- **HF-Space-ready API** — `src/api.py` is a pure interface for a Gradio app;
  scaffold in `app/`.

## Quick start

```bash
# 0. Set up environment
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# 1. (Optional) Re-download prices — committed copy already in data/prices/
python scripts/01_download_data.py

# 2. (Optional) Re-run Monte Carlo — committed copy already in data/mc_cache/
```

## Common workflows

### Add a new ticker

1. Edit `config/settings.py`: add the ticker to `UNIVERSE` with its segment
   and display name.
2. Re-download prices:
   ```bash
   python scripts/01_download_data.py --force
   ```
3. Run Monte Carlo for the new ticker only:
   ```bash
   python scripts/02_run_montecarlo.py
   ```
4. Run all analyses for the new ticker:
   ```bash
   python scripts/03_run_analysis.py --methods all --tickers NEWTKR
   ```

### Re-run SV-ADF on a different window

```bash
python scripts/03_run_analysis.py --methods svadf \
    --svadf-start 2018-01-01 --svadf-end 2022-12-31 --force
```

Results from the prior window are kept side-by-side in `data/results/svadf/`.

### Re-run GSADF after changing episode filters

Edit `EPISODE_MIN_DURATION` and `EPISODE_MERGE_GAP` in `config/settings.py`,
then:

```bash
python scripts/03_run_analysis.py --methods gsadf --force
```

The MC cache is reused; only the episode-dating step re-runs.

### Regenerate a single figure type

```bash
# All GSADF segment panels only
python scripts/05_generate_figures.py --types gsadf_panel

# Comparison overlays for specific tickers
python scripts/05_generate_figures.py --tickers NVDA MSFT --types gsadf_vs_svadf
```

### Resume after a crash

Re-run the same script. Skip-if-exists is on by default everywhere
(use `--force` to override).

## Spurious-bubble filters

`config/settings.py` defaults:

| Parameter | Default | Effect |
|---|---|---|
| `EPISODE_MIN_DURATION` | 21 days | Drop episodes shorter than this |
| `EPISODE_MERGE_GAP` | 21 days | Merge episodes separated by ≤ this gap |

Applied in `src/methods/episodes.detect_bsadf_episodes` after raw BSADF
crossing detection. Tune both in settings without re-running Monte Carlo.

## Key findings (thesis)

- **SV-ADF W1** (post-ChatGPT: Nov 2021 – May 2026) detects explosive episodes
  in 11 equity tickers across semiconductor, hyperscale, and application-AI
  segments; zero detections in the W2 pre-ChatGPT window (the AI bubble is a
  post-ChatGPT phenomenon).
- **GSADF** (PSY 2015) rejects the global null for the majority of AI-segment
  tickers and date-stamps multiple distinct episodes, with episode density
  concentrated post-Q4 2023.
- **Commodity evidence (H6):** GSADF rejects for all commodity groups, but
  post-Q4-2023 episode timing and SV-ADF W1 confirmation is confined to
  precious metals (GLD, SLV) — AI-relevant energy and broad benchmarks show
  no SV-ADF confirmation.
